# LaunchDarkly Feature Flags for the Operations Phase

This guide extends the AWS AI-DLC workshop Operations phase with LaunchDarkly feature flags, enabling progressive delivery and instant feature control for the AnyCompanyRead e-commerce application.

## Overview

### Why Feature Flags for E-commerce?

The AnyCompanyRead application generated by the AI-DLC workshop is a production-like e-commerce system. Feature flags provide:

- **Progressive rollouts** - Deploy new checkout flow to 10% of users first
- **Instant rollbacks** - Disable problematic features in seconds, not deployments
- **A/B testing** - Compare conversion rates between UI variations
- **User segmentation** - Premium features for paying customers
- **Operational control** - Kill switches for resource-intensive features

### What You'll Build

1. Add the LaunchDarkly SDK to the AnyCompanyRead frontend and backend
2. Create feature flags for key functionality
3. Implement flag-controlled features
4. Demo toggling flags and seeing instant UI changes

## Step 1: Add LaunchDarkly SDK

### Frontend (React)

```bash
npm install launchdarkly-react-client-sdk
```

Configure the provider in your app entry point:

```tsx
// src/index.tsx or src/App.tsx
import { LDProvider } from 'launchdarkly-react-client-sdk';

const LD_CLIENT_SIDE_ID = process.env.REACT_APP_LD_CLIENT_SIDE_ID;

function App() {
  // Get user from your auth system
  const user = useAuth();

  const ldContext = {
    kind: 'user',
    key: user?.id || 'anonymous',
    email: user?.email,
    custom: {
      plan: user?.plan || 'free',
    },
  };

  return (
    <LDProvider
      clientSideID={LD_CLIENT_SIDE_ID}
      context={ldContext}
      options={{ bootstrap: 'localStorage' }}
    >
      <Router>
        {/* Your app routes */}
      </Router>
    </LDProvider>
  );
}
```

### Backend (Node.js/Express)

```bash
npm install @launchdarkly/node-server-sdk
```

Initialize in your server:

```javascript
// src/server.js or src/app.js
const LaunchDarkly = require('@launchdarkly/node-server-sdk');

const ldClient = LaunchDarkly.init(process.env.LAUNCHDARKLY_SDK_KEY);

// Wait for initialization before handling requests
ldClient.waitForInitialization().then(() => {
  console.log('LaunchDarkly initialized');
  app.listen(PORT);
});

// Make client available to routes
app.use((req, res, next) => {
  req.ldClient = ldClient;
  next();
});
```

### Backend (Python/FastAPI)

```bash
pip install launchdarkly-server-sdk
```

```python
# main.py
from ldclient import LDClient, Config
from fastapi import FastAPI, Request

app = FastAPI()
ld_client = LDClient(Config(os.environ["LAUNCHDARKLY_SDK_KEY"]))

@app.middleware("http")
async def add_ld_client(request: Request, call_next):
    request.state.ld_client = ld_client
    return await call_next(request)
```

## Step 2: Create Feature Flags

Create these flags in your LaunchDarkly project:

### Flag 1: new-checkout-flow

**Purpose**: Progressive rollout of redesigned checkout experience

| Property | Value |
|----------|-------|
| Key | `new-checkout-flow` |
| Type | Boolean |
| Description | Enable streamlined single-page checkout |
| Default (Off) | `false` |
| Default (On) | `true` |

**Targeting Strategy**:
- Start at 0% (flag off)
- Rollout: 10% → 25% → 50% → 100%
- Monitor conversion metrics at each stage

### Flag 2: premium-features

**Purpose**: Unlock premium functionality for paying customers

| Property | Value |
|----------|-------|
| Key | `premium-features` |
| Type | Boolean |
| Description | Enable premium member features |
| Default (Off) | `false` |
| Default (On) | `true` |

**Targeting Strategy**:
- Target users where `plan` is `pro` or `enterprise`
- Default off for free users

### Flag 3: ai-recommendations

**Purpose**: Control AI-powered product recommendations

| Property | Value |
|----------|-------|
| Key | `ai-recommendations` |
| Type | Boolean |
| Description | Show AI-powered product recommendations |
| Default (Off) | `false` |
| Default (On) | `true` |

**Targeting Strategy**:
- Percentage rollout starting at 10%
- Kill switch for high-load scenarios

### Flag 4: checkout-button-style (A/B Test)

**Purpose**: Test different checkout button designs

| Property | Value |
|----------|-------|
| Key | `checkout-button-style` |
| Type | String |
| Variations | `default`, `prominent`, `minimal` |
| Description | A/B test checkout button styles |

**Targeting Strategy**:
- Equal split: 33% each variation
- Measure click-through and conversion rates

## Step 3: Implement Flag-Controlled Features

### Frontend: Checkout Component

```tsx
// src/components/Checkout.tsx
import { useFlags, useLDClient } from 'launchdarkly-react-client-sdk';

export function Checkout({ items, onComplete }) {
  const { newCheckoutFlow, checkoutButtonStyle } = useFlags();
  const ldClient = useLDClient();

  // Track for experimentation
  const handleCheckoutStart = () => {
    ldClient?.track('checkout-started', { itemCount: items.length });
  };

  if (newCheckoutFlow) {
    return (
      <NewCheckout
        items={items}
        buttonStyle={checkoutButtonStyle}
        onStart={handleCheckoutStart}
        onComplete={onComplete}
      />
    );
  }

  return (
    <LegacyCheckout
      items={items}
      onStart={handleCheckoutStart}
      onComplete={onComplete}
    />
  );
}
```

### Frontend: Premium Features

```tsx
// src/components/PremiumBadge.tsx
import { useFlags } from 'launchdarkly-react-client-sdk';

export function PremiumBadge() {
  const { premiumFeatures } = useFlags();

  if (!premiumFeatures) return null;

  return <span className="premium-badge">⭐ Premium</span>;
}

// src/components/ProductPage.tsx
export function ProductPage({ product }) {
  const { premiumFeatures, aiRecommendations } = useFlags();

  return (
    <div>
      <ProductDetails product={product} />

      {premiumFeatures && (
        <PremiumPricing product={product} />
      )}

      {aiRecommendations && (
        <AIRecommendations productId={product.id} />
      )}
    </div>
  );
}
```

### Backend: API Feature Control

```javascript
// routes/recommendations.js
router.get('/recommendations', async (req, res) => {
  const context = {
    kind: 'user',
    key: req.user?.id || 'anonymous',
  };

  const aiEnabled = await req.ldClient.variation(
    'ai-recommendations',
    context,
    false  // default
  );

  if (!aiEnabled) {
    // Return static recommendations
    return res.json({ products: getStaticRecommendations() });
  }

  // Call AI recommendation service
  const recommendations = await getAIRecommendations(req.user);
  res.json({ products: recommendations });
});
```

```python
# Python equivalent
@app.get("/recommendations")
async def get_recommendations(request: Request):
    context = {
        "kind": "user",
        "key": request.state.user_id or "anonymous"
    }

    ai_enabled = request.state.ld_client.variation(
        "ai-recommendations",
        context,
        False
    )

    if not ai_enabled:
        return {"products": get_static_recommendations()}

    return {"products": await get_ai_recommendations(request.state.user_id)}
```

## Best Practices

### Naming Conventions

- Use kebab-case: `new-checkout-flow`, not `newCheckoutFlow`
- Be descriptive: `premium-pricing-display`, not `flag-1`
- Group related flags: `checkout-*`, `premium-*`

### Default Values

Always provide sensible defaults in code:

```tsx
const { newCheckoutFlow = false } = useFlags();
```

### Context Best Practices

Include relevant attributes for targeting:

```tsx
const context = {
  kind: 'multi',
  user: {
    key: user.id,
    email: user.email,
    plan: user.plan,  // for premium targeting
  },
  device: {
    key: deviceId,
    platform: 'web',  // for platform-specific features
  }
};
```

### Flag Lifecycle

1. **Create** flag with targeting rules
2. **Deploy** code that checks the flag
3. **Enable** flag progressively
4. **Monitor** metrics and user feedback
5. **Clean up** - remove flag after full rollout

## Troubleshooting

### Flag Not Evaluating

```tsx
// Debug flag evaluation
const ldClient = useLDClient();
const value = ldClient?.variation('my-flag', false);
console.log('Flag value:', value);
console.log('Context:', ldClient?.getContext());
```

### SDK Not Initializing

- Verify the client-side ID (frontend) or SDK key (backend)
- Check network requests in browser DevTools
- Ensure environment matches (dev vs prod)

### Stale Values

```tsx
// Force refresh flags
ldClient?.identify(newContext);
```
